# System Design Exercises

Structured system design practice. Each exercise: draft → evaluate → revise.

## Structure

```
exercises/
└── 001/
    ├── problem.md   # constraints and requirements
    └── design.md    # architecture, tradeoffs, review notes
rubric.md            # evaluation criteria
```

## Commands

- `/new` — generate exercise
- `/grade` — evaluate design, suggest revisions
- `/review` — progress report across exercises
